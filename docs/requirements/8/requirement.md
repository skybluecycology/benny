# KG3D-001 — Requirement Specification

**Document status:** Normative. Overrides any conflicting prose in
the source brief. Every requirement below is uniquely addressable by
its ID and is verified by at least one test listed in
[acceptance_matrix.md](acceptance_matrix.md).

## 1. Scope

Build, inside the existing Benny repo, a **3D Machine Learning
Knowledge Graph cognitive mesh** that:

1. Loads a canonical ML concept ontology (≥ 2081 nodes, ≥ 5149 edges,
   matching the `cosmic-cortex/ml-knowledge-graph` baseline).
2. Renders that graph in 3D inside Benny Studio (V2 "God-Mode" shell)
   with force-directed physics, hierarchical AoT layout, and
   Focus+Context interactions.
3. Scales to 100 000 instanced nodes at ≥ 60 fps on a reference mid
   tier laptop (see [§6 Non-functional targets](#6-non-functional-targets)).
4. Is **bidirectionally** wired to the Studio code editor so AST
   detections focus the relevant concept node.
5. Supports optional **WebXR** embodied exploration with a
   World-in-Miniature overview.
6. Grows automatically via an **LLM Text-to-Cypher + GCoT** ingestion
   pipeline with strict HITL approval.
7. Ships behind a feature flag (`ui.kg3d_enabled`, default **off**) so
   the existing V1/V2 experience is never regressed.

Out of scope for KG3D-001: replacing the existing 2D workflow canvas,
migrating Neo4j to another store, or authoring educational content.

## 4. Data contracts (normative)

All schemas live in `benny/graph/kg3d/schema.py` and are mirrored in
`frontend/src/types/kg3d.ts`. The two files **must** be generated by
the same source (see [implementation_plan.md](implementation_plan.md)
Phase 0) and a test enforces equivalence.

### 4.1 Node

```json
{
  "id": "string (stable hash of canonical_name)",
  "canonical_name": "string, unique",
  "display_name": "string",
  "category": "enum (see §4.3)",
  "aot_layer": "integer, 1 ≤ layer ≤ 5 (1 = most abstract)",
  "metrics": {
    "pagerank": "float ≥ 0",
    "degree": "integer ≥ 0",
    "betweenness": "float ≥ 0",
    "descendant_ratio": "float 0..1",
    "prerequisite_ratio": "float 0..1",
    "reachability_ratio": "float 0..1"
  },
  "position_hint": { "x": "float", "y": "float", "z": "float" } | null,
  "source_refs": ["string (document id or URL)"],
  "created_at": "ISO-8601 UTC",
  "updated_at": "ISO-8601 UTC"
}
```

Invariants (enforced by `kg3d.schema.validate_node`):

- `aot_layer` is derived from `descendant_ratio` per the binning in
  `schema.aot_layer_for(descendant_ratio)`; it may not be set directly
  by callers other than the ingest pipeline.
- All metric floats are finite (no NaN, no Inf).
- `canonical_name` is globally unique (case-insensitive).

### 4.2 Edge

```json
{
  "id": "string (uuid v4)",
  "source_id": "string (node.id)",
  "target_id": "string (node.id)",
  "kind": "enum { prerequisite, references, contradicts, generalises, specialises }",
  "weight": "float > 0",
  "evidence": ["string"],
  "created_at": "ISO-8601 UTC"
}
```

Invariants:

- `source_id != target_id` (no self-loops at ingest time; visual
  self-loops are a renderer concern, not a data concern).
- The subgraph restricted to `kind == "prerequisite"` is a DAG (checked
  by `test_kg3d_prerequisite_dag_invariant`).

### 4.3 Category enumeration

Exactly the sixteen categories enumerated in the source brief; the
enum is frozen at Phase 0:

```
approximations_expansions, ai_deep_learning, calc_variations_control,
combinatorics_number_theory, computer_vision_pattern_recognition,
functional_analysis_real_functions, information_communication_theory,
llm_nlp, linear_multilinear_algebra_matrix_theory,
measure_integration, neural_evolutionary_computing,
numerical_analysis_signal_processing, ops_research_math_programming,
optimisation_reinforcement_learning, ode_pde,
probability_stochastic_statistics
```

### 4.4 Delta stream (SSE)

`GET /api/kg3d/stream` emits Server-Sent-Events conforming to:

```json
{ "kind": "upsert_node" | "upsert_edge" | "remove_node" | "remove_edge" |
          "metrics_refresh" | "heartbeat",
  "payload": "…matching §4.1 / §4.2…",
  "seq": "monotonic integer",
  "ts": "ISO-8601 UTC" }
```

The client MUST tolerate reorder within a 2 s window and MUST drop
events with a `seq` lower than the last applied value.

## 5. Functional requirements

Each requirement below has the form `KG3D-F{N}` and is covered by at
least one test of the form `test_kg3d_f{n}_*` in the Python or
frontend suite.

### 5.1 Ontology & storage

- **KG3D-F1** — The backend ships a canonical ontology loader
  `benny/graph/kg3d/ontology.py::load_default_ontology()` that returns a
  `Graph` with **exactly** the number of nodes and edges reported by
  the bundled fixture `tests/fixtures/kg3d/ml_knowledge_graph_v1.json`,
  which itself reproduces the 2081/5149 baseline to byte-equality.
- **KG3D-F2** — All six graph metrics in §4.1 are computed by
  `kg3d.metrics.compute_all(graph)` and cached in a SQLite artefact at
  `workspace/.benny/kg3d/metrics.sqlite`. Cache is invalidated when the
  ontology's content hash changes.
- **KG3D-F3** — Storage uses the existing Neo4j driver
  (`benny.core.graph_db.get_driver`). A new label `:MLConcept` is added;
  no existing labels are modified. A migration script
  `benny/graph/kg3d/migrations/001_create_mlconcept.cypher` is
  idempotent (safe to run twice).
- **KG3D-F4** — `GET /api/kg3d/ontology` returns the full graph as
  §4.1/§4.2 JSON. Response size ≤ 8 MB gzipped for the default
  ontology.

### 5.2 Rendering (baseline 3D)

- **KG3D-F5** — Frontend component
  `frontend/src/components/Studio/kg3d/SynopticWeb.tsx` renders the
  ontology using `3d-force-graph` over Three.js. The component is
  mounted only when `uiVersion === 'v2'` **and** `ui.kg3d_enabled`.
- **KG3D-F6** — Node visual attributes are derived purely from §4.1:
  - volumetric radius ∝ `log(1 + degree)`,
  - colour mapped from `category` via a fixed sixteen-entry palette in
    `frontend/src/components/Studio/kg3d/palette.ts`,
  - emissive intensity ∝ `pagerank`,
  - Y-axis position = `(k - aot_layer) * AOT_Y_GAP` (default 12).
- **KG3D-F7** — Edges are directed and rendered with animated particle
  emission (`emitParticle`) when `kind == "prerequisite"`.
- **KG3D-F8** — Layout pre-warms with `warmupTicks = 120` and cools
  with `cooldownTicks = 400`; adding/removing nodes reheats via
  `d3ReheatSimulation()`.

### 5.3 Scalability

- **KG3D-F9** — When `nodes.length ≥ INSTANCE_THRESHOLD` (default 2500,
  override via `ui.kg3d.instance_threshold`), rendering switches to a
  `THREE.InstancedMesh` path in
  `frontend/src/components/Studio/kg3d/InstancedNodes.tsx`.
- **KG3D-F10** — The instanced path uses a single draw call for nodes
  and a second single draw call for edges. Measured via
  `renderer.info.render.calls ≤ 100` during steady state interaction.
- **KG3D-F11** — Physics runs in a dedicated Web Worker
  (`frontend/src/workers/kg3d_physics.worker.ts`). Positions are posted
  to the main thread as `Float32Array` transferables at ≤ 30 Hz.
- **KG3D-F12** — Low-importance nodes (`degree ≤ 2` and distance to
  camera > `LOD_FAR`) degrade to `THREE.Points` sprites. This is
  reversible as the camera approaches.

### 5.4 Cognitive structuring

- **KG3D-F13** — The AoT hierarchy is rendered as a five-tier vertical
  stratification (layer 1 highest). Tier boundaries are drawn as
  translucent Peircean cuts (`CutLayer.tsx`) and correspond exactly to
  node `aot_layer` values.
- **KG3D-F14** — A Prolog-style constraint checker
  `benny/graph/kg3d/constraints.py` fails to publish a layout if any
  prerequisite edge points from a lower `aot_layer` to a strictly
  higher one (i.e. a prerequisite cannot be more abstract than its
  dependent).
- **KG3D-F15** — Focus+Context: selecting a node triggers
  `focusPath(nodeId)` which dims (α = 0.08, greyscale) all nodes not on
  the transitive prerequisite closure of the selected node.
- **KG3D-F16** — A "Semantic zoom" step replaces clusters of ≥ 25
  sibling nodes whose bounding sphere is smaller than 48 px on screen
  with a single **prototype node** labelled by the shared category;
  zooming past the threshold expands the cluster again. No data
  mutation occurs — this is purely a view transformation.

### 5.5 Studio integration

- **KG3D-F17** — A new Zustand slice
  `frontend/src/hooks/useKg3dStore.ts` exposes `{ nodes, edges, focus,
  setFocus, mode, setMode }`. It is hydrated from `/api/kg3d/ontology`
  on mount.
- **KG3D-F18** — The existing Monaco IPC bus emits a
  `{ kind: "symbol_detected", symbol: string }` event on AST matches.
  `SynopticWeb.tsx` subscribes and calls `setFocus(symbol)` only if a
  node with matching `canonical_name` (case-insensitive) exists;
  otherwise it is a no-op (no error, no toast).
- **KG3D-F19** — Camera transitions to the focus node using an
  ease-in-out cubic of duration `600 ms ± 20 ms`.
- **KG3D-F20** — When the SynopticWeb panel is closed
  (`onDidDispose`), the renderer, Web Worker, and SSE stream are torn
  down within 250 ms; `renderer.info.memory.textures == 0` post-close.

### 5.6 WebXR (optional tier)

- **KG3D-F21** — A `SynopticWebXR.tsx` component starts a WebXR session
  when available; its absence is a silent feature skip (no UI error).
- **KG3D-F22** — WIM is a 1:20 replica anchored at
  `player.chest + (0, -0.35 m, -0.45 m)`; pointing at a WIM node
  teleports the user's head to the full-scale equivalent.
- **KG3D-F23** — Controller raycasts can grab, drag, and release nodes
  without mutating the underlying ontology (visual-only displacement).

### 5.7 LLM-driven ingestion

- **KG3D-F24** — `benny/graph/kg3d/ingest.py::propose_from_document(doc)`
  returns a `Proposal { nodes_upsert, edges_upsert, rationale_md }`.
  The function MUST run through `benny.core.llm_router` so
  `BENNY_OFFLINE=1` yields `OfflineRefusal`.
- **KG3D-F25** — A Grounded-Chain-of-Thought validator
  `kg3d.gcot.validate(proposal, existing_graph)` checks: no DAG
  violation, no unknown category, no duplicate `canonical_name`,
  metrics schema compliance. Failing proposals are rejected with
  machine-readable reasons.
- **KG3D-F26** — Accepted proposals are persisted only after explicit
  HITL approval via `POST /api/kg3d/proposals/{id}/approve`. Rejection
  is `POST /api/kg3d/proposals/{id}/reject` and records the reviewer's
  note.
- **KG3D-F27** — The SSE delta stream (§4.4) emits the resulting
  upserts within 1 s of approval.

## 8. Security & privacy

- **KG3D-SEC1** — All Webview loads go through `asWebviewUri` with
  `localResourceRoots` restricted to `frontend/dist` and the ontology
  fixture directory.
- **KG3D-SEC2** — CSP header injected into the SynopticWeb HTML:
  `default-src 'none'; script-src ${cspNonce}; style-src 'unsafe-inline';
   img-src data: vscode-webview-resource:; connect-src ${serverUrl};`.
- **KG3D-SEC3** — LLM-proposed Cypher is **never** executed directly.
  Proposals are transformed into parameterised MERGE statements by
  `kg3d.ingest.to_cypher(proposal)`; string interpolation into Cypher
  is forbidden and linted by `tests/safety/test_kg3d_no_cypher_interp.py`.
- **KG3D-SEC4** — No PII flows through the ingest pipeline; documents
  are truncated to ≤ 512 KB before LLM call.

## 9. Observability

- **KG3D-OBS1** — `benny doctor --json` gains a `kg3d` section
  reporting: ontology hash, node/edge counts, metrics-cache freshness,
  SSE stream health, last ingest proposal timestamp.
- **KG3D-OBS2** — Structured logs emitted by `benny/graph/kg3d/**`
  carry `component="kg3d"` and follow the Phase 6 LLM log schema.
- **KG3D-OBS3** — Frontend emits `perf.kg3d.fps`, `perf.kg3d.draw_calls`
  to the existing telemetry channel once per second while the panel is
  open.

## 10. Rollback

Every phase ships behind `ui.kg3d_enabled=false`. Rolling back =
reverting the merge commit for that phase; no schema migrations are
destructive (all Neo4j migrations are additive and idempotent — see
KG3D-F3).
